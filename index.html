Abstract

Current popular weather services in Ontario provide the user with weather information corresponding to the closest weather station. Unfortunately, as is often the case, the nearest weather station to a given location can be many kilometers away. When this is the case, the user’s weather information can be vastly different from reality. To fix this problem, Owen Andrews-Simons of Arubrum Innovations has asked our group to develop a product that can estimate the actual weather at the user’s exact location (or chosen location). Additionally, we were tasked to provide a new report every hour using only open-source technologies and solutions (no-paid licenses). To accomplish this, our team has developed an interpolation model in QGIS, a robust extract transfer and load (ETL) pre-process workflow using python, and a web map weather report that utilizes Leaflet. Our methodologies used were (1) data acquisition and pre-processing, (2) database design, (3) spatial and statistical analysis, (4) programming/customization, and (5) visualization. Our project has met our client’s expectations of delivering an automated location-specific weather web report of Ontario while using open-source technology.
